# 业务规则清单 - AI代理编写测试的输入源

## 使用说明

1. AI代理编写测试前，必须先阅读此清单
2. 每个测试文件必须至少覆盖1条业务规则
3. 每个业务规则必须有正向、负向场景测试

## 业务规则

### BR-01: ContextAwareTransformer匹配成功后必须设置patched=True

- **代码依据**: `transformer.py`
- **位置**: 第53行
- **说明**: 当函数名和类名都匹配时，必须设置self.patched=True，否则GovernanceExecutor会误判为'未找到目标'

### BR-02: ContextAwareTransformer必须精确匹配目标类中的方法

- **代码依据**: `transformer.py`
- **位置**: 第47行
- **说明**: class_match条件为(target_class is None or current_class == target_class)，确保只修改指定类中的方法

### BR-03: FunctionTransformer匹配成功后必须设置patched=True

- **代码依据**: `transformer.py`
- **位置**: 第28行
- **说明**: 当函数名匹配时，必须设置self.patched=True

### BR-04: ImportApplier必须在第一个import语句后插入新import

- **代码依据**: `transformer.py`
- **位置**: 第75行
- **说明**: 遍历模块体，在遇到第一个import时插入新import语句

### BR-05: ImportApplier在无现有import时必须在文件开头插入

- **代码依据**: `transformer.py`
- **位置**: 第85行
- **说明**: 如果没有找到任何import语句，必须在模块体开头插入新import

### BR-06: GovernanceExecutor在应用补丁前必须验证patched标志

- **代码依据**: `executor.py`
- **位置**: 第169行
- **说明**: 如果transformer.patched为False，必须抛出RuntimeError表示'未找到目标'

### BR-07: GovernanceExecutor必须使用安全验证器验证补丁

- **代码依据**: `executor.py`
- **位置**: 第47行
- **说明**: 应用补丁前必须通过SecurePathValidator验证路径安全性

### BR-08: ExecutionPipeline必须在断言失败后执行治理处理器

- **代码依据**: `pipeline.py`
- **位置**: 第29行
- **说明**: 当断言处理器返回失败时，必须继续执行GovernanceProcessor进行治理

### BR-09: ExecutionPipeline必须正确识别GovernanceProcessor

- **代码依据**: `pipeline.py`
- **位置**: 第31行
- **说明**: 通过processor.__class__.__name__ == 'GovernanceProcessor'判断是否为治理处理器

### BR-10: ApprovalManager必须只批准安全和重构类型的补丁

- **代码依据**: `approval.py`
- **位置**: 第35行
- **说明**: 只有patch_type为'security'或'refactoring'时才能通过审批

### BR-11: GovernanceOrchestrator必须按顺序执行治理流程

- **代码依据**: `orchestrator.py`
- **位置**: 第50行
- **说明**: 执行顺序：诊断→审批→补丁→追踪→监控

### BR-12: GovernanceOrchestrator必须验证诊断报告非空

- **代码依据**: `orchestrator.py`
- **位置**: 第195行
- **说明**: 应用补丁前必须确保diagnosis_report不为空

### BR-13: Registry必须注册governance处理器

- **代码依据**: `registry.py`
- **位置**: 第15行
- **说明**: _PROCESSOR_MAP必须包含'governance'到GovernanceProcessor的映射

### BR-14: Worker任务异常处理必须通过GovernanceOrchestrator触发治理流程

- **代码依据**: `tasks.py`
- **位置**: 第85行
- **说明**: 不能直接调用AIGovernanceAgent，必须走完整治理闭环

### BR-15: 默认pipeline配置必须包含governance处理器

- **代码依据**: `tasks.py`
- **位置**: 第45行
- **说明**: DEFAULT_PIPELINE_CONFIG必须包含'governance'步骤

- 规则ID范围: BR-01 ~ BR-15
